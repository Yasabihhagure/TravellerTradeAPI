# ソフトウェア仕様書：Traveller Trade API (TTA)

## 1. 概要

本プロジェクトは、TRPG『Traveller (Mongoose 2nd Edition / Update 2022)』における星間貿易を支援するためのウェブツールである。

GitHub Pagesでホストされ、人間だけでなく**GeminiやChatGPTなどのAIエージェントが「外部ツール」として利用し、ユーザーに最適な貿易ルートを回答できること**を主眼に設計する。

## 2. 設計指針

1. **動的なデータ取得** : 星系データ（UWP、貿易コード等）をハードコードせず、実行時に [Traveller Map API](https://travellermap.com/doc/api) から取得する。
2. **著作権への配慮** : セクターデータそのものは保持せず、アップロードされた「Update 2022」の貿易ルール（価格修正ロジック等）のみをアルゴリズムとして実装する。
3. **AIフレンドリー** : AIがブラウジング機能でアクセスした際、即座に機械判読可能な形式（JSON）でデータを抽出できる構造にする。

## 3. 機能要件

### 3.1 貿易計算エンジン

* **入力** : 出発地（Sector/Hex）、目的地（Sector/Hex）、プレイヤーの〈ブローカー〉技能レベル。
* **ロジック** :
* 両地点の貿易コード（Ag, In, Ni等）を比較。
* 『星間貿易品価格・在庫一覧表』に基づき、出発地で購入DMが高く、目的地で売却DMが高い商品を特定。
* 『投機的貿易修正価格表』に基づき、期待される純利益率を算出。
* **出力** : 推奨商品リスト、期待利益、各星系のスペック。

### 3.2 AI中継インターフェース (The AI Bridge)

* **URLルーティング** : AIが直接アクセスしやすいよう、フラグメントまたはクエリパラメータによるルート指定をサポートする。
* 例: `index.html#/trade/Spinward-Marches/2124/2526`
* **データ埋め込み** : 指定されたルートの計算結果を、HTML内の特定の `id` を持つ `<script>` タグ（JSON形式）として動的に生成・描画する。

## 4. インターフェース仕様（AI向け応答形式）

AIがページを読み込んだ際、以下のJSONデータがページ内に存在することを保証する。

**JSON**

```
{
  "api_version": "1.0.0",
  "status": "ready",
  "route_analysis": {
    "origin": {
      "name": "Lunion",
      "coords": "2124",
      "uwp": "A995984-D",
      "trade_codes": ["Hi", "In", "Na", "Va"]
    },
    "destination": {
      "name": "Shirene",
      "coords": "2526",
      "uwp": "A437424-C",
      "trade_codes": ["Ag", "Ga", "Ni"]
    },
    "recommendations": [
      {
        "item": "Industrial Electronics",
        "purchase_dm": 2,
        "sale_dm": 2,
        "net_profit_potential": "Very High",
        "logic": "Bought on Industrial (In) world, sold on Non-Industrial (Ni) world."
      }
    ]
  }
}
```
