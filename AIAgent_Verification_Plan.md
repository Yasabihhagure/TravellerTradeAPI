# ChatGPT (LLM) 連携検証項目

本ツール (Traveller Trade API) を実際の環境（GitHub Pages 等）にデプロイした後、ターゲットユーザーである ChatGPT (Web Browsing 機能等を持つ LLM) が、仕様通りにツールを使いこなせるかを確認するための検証手順とプロンプト集です。

## 推奨検証環境
*   **対象LLM**: ChatGPT (GPT-4 / GPT-4o 等、ウェブブラウジング機能が有効なモデル)
*   **デプロイ先**: `[あなたのデプロイ先 URL]` (例: `https://[username].github.io/TravellerTradeAPI/`)

---

## 検証フェーズ1: ツールの存在と仕様の理解

**目的**: LLM が `README.md` またはトップページから、本ツールの使い方（URLパラメータ、JSONの読み取りなど）を正しく学習できるか確認する。

**ユーザープロンプト例**:
> 以下のURLに、Traveller TRPG 用の貿易計算ツールがあります。このツールの使い方や、AIエージェントであるあなたがどのようにこのツールを使ってデータを得るべきか、仕組みを要約してください。
> URL: `[デプロイ先URL]`

**✅ 期待される結果**:
*   入力パラメータ（Sector, Origin Hex, Destination Hex, Broker Level）の指定方法を理解している。
*   URLパラメータ（`#/trade/[Sector]/[Origin]/[Dest]?broker=[Lv]`）を使って直接検索を実行できることを理解している。
*   HTML内から `<script id="tta-data">` を探し、中のJSONから結果を抽出する仕組みを理解している。
*   （オプション）目的地が不明な場合、TravellerMap API の `/api/jump` 等を使って周辺星系を探す手順を理解している。

---

## 検証フェーズ2: 基本的な貿易ルートの計算と解釈

**目的**: 座標が明確に指定された基本的なシナリオで、LLMがツールを実行し、結果を正しく人間に提示できるか確認する。

**ユーザープロンプト例**:
> Spinward Marches セクターの Lunion (2124) から Shirene (2125) へ向かおうとしています。貨物室に空きがあるので貿易をしたいです。私のキャラクターの Broker 技能はレベル 1 です。
> 先ほどのツールを使って、最も利益が出そうな商品を3つ教えてください。また、なぜそれが選ばれたのか（DMの理由など）もツールから読み取って解説してください。

**✅ 期待される結果**:
*   LLM が自律的に `[デプロイ先URL]/#/trade/Spinward-Marches/2124/2125?broker=1` のようなURLを生成し、ブラウジングアクセスを実行する。
*   `tta-data` から上位3つの商品を正確に抽出する（例: ロボット、サイバネティクス 等）。
*   「Lunionの特性（〇〇）により購入DMが+〇、Shireneの特性（〇〇）により売却DMが+〇だから有利です」といった理由を、得られたデータに基づき適切に解説する。

---

## 検証フェーズ3: 応用シナリオ (周辺星系の自律検索)

**目的**: 目的地が不明な場合、LLMがツールとTravellerMap APIを組み合わせて自律的に最適な行動を取れるか確認する。

**ユーザープロンプト例**:
> 今、Spinward Marches の Regina (1910) にいます。ジャンプ-2 以内の範囲で、最も儲かりそうな貿易ルート（行き先と商品）を1つ探して、プランを提案してください。Broker 技能はありません。

**✅ 期待される結果**:
*   LLM が TravellerMap API (`https://travellermap.com/api/jump?sector=Spinward%20Marches&hex=1910&jump=2`)等を使用して、Regina からの移動可能範囲の Hex 一覧を取得する。
*   取得した近隣の惑星いくつかに対して、本ツール（`[デプロイ先URL]/#/trade/...`）を複数回呼び出して利益を比較検討する（または代表的な有力候補をシミュレートする）。
*   最も利益の高いルートを提示し、推奨商品と予想利益、可能であれば TravellerMap の画像URLを使って星図を提示する。

---

## 検証フェーズ4: 異常系 (エラーフェイルセーフ)

**目的**: 存在しない座標や誤ったセクター名を指定した場合の、LLMのフォールバック動作を確認する。

**ユーザープロンプト例**:
> Spinward Marches の Earth (0000) から Mars (0001) への貿易ルートを計算してください。

**✅ 期待される結果**:
*   LLM がツールにアクセスするが、ツール側でエラー（"World not found" 等）が発生する。
*   LLM がエラー内容を正確に把握し、「指定された座標 (0000 や 0001) は TravellerMap 上に存在しないため、計算できませんでした。正しい座標または星系名を教えてください」と適切にユーザーにフィードバックを返す。

---
**テスト完了条件**: すべてのフェーズで、ChatGPT が幻覚（ハルシネーション）による適当な返答ではなく、**実際にツールの `tta-data` または TravellerMap API から取得した「事実」に基づいて** 返答を生成できれば検証成功となります。
